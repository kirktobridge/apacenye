# W1-v0 — Forecast-anchored fair value (NYC daily high temperature)

**Status:** implemented (Stage 5 reference worker) · **Mode:** PAPER only

**Thesis.** Retail flow anchors on the point forecast ("the app says 86°");
the correct object is a probability distribution over the day's high, so
brackets near the point forecast are systematically overpriced and tails
mispriced. Persists because attention is scarce in small daily markets.
(Assume the easy mispricings are picked over by other bots — Stage 1 §3.1.)

**Data sources.** Signal: NWS public forecast API for the KNYC grid point
(OKX 34,45 — verified 2026-07-19), free, keyless, `User-Agent` required.
Settlement: NWS Climatological Report (Daily) for Central Park — the market's
own rules source. Markets: Kalshi series `KXHIGHNY` (verified live; tail
brackets are STRICT inequalities — see `catalog.py`).

**p_model.** `T_max ~ Normal(μ = forecast high, σ = 3.2°F)`; bracket
probability via CDF with ±0.5° continuity correction
(`domain/weather.py::bracket_probability`). σ is evidence-based (OD-11
resolved 2026-07-19): population std of GFS-extended-MOS daytime-max forecast
error vs observed KNYC highs at the shortest archived lead (≤48h) — raw 3.14,
n=183, bias −0.64°F — rounded up to 3.2 (conservative; MEX proxies the live
NDFD signal and its shortest lead is ~36h, so this is an upper bound for the
same-day horizon). Study + provenance in `research/estimate_sigma_w1.py`.
Intraday σ tightening below this is deferred to shadow-forecast calibration
(OD-9), the sanctioned path for tuning σ.

**Entry/exit.** Evaluate every bracket on 10-minute ticks; propose the better
of YES-at-ask / NO-at-mirror when `net_edge ≥ 0.04` after fee + 1¢ slippage.
Hold to settlement (no exit fee; capital frees nightly). Cancel outstanding
intents when the forecast goes stale (> 12 h) or the edge flips.

**Sizing.** λ=0.5 shrinkage → quarter-Kelly → hard caps (event 5%, strategy
20%, portfolio 50%, ≤25% of depth, ≤100 contracts). Caps always dominate.

**Worst case (paper).** All brackets of one event are ONE exposure: a bad
forecast day loses at most the $50 event cap; the strategy's cap bounds total
loss at $200 (20% of $1,000). Cross-day/cross-city correlation is mitigated
only by the portfolio cap in v0 (OD-20).

**Open Decisions this strategy depends on:** OD-1 (fee schedule per series —
general rate verified, series-specific not), OD-2 (real spreads/depth — gate:
spread ≤ 2¢). OD-11 (σ archive) RESOLVED 2026-07-19; OD-19 (auth) and
OD-3-for-NYC verified.

**Upgrade path.** W1-v1 swaps the Gaussian for ensemble output (GEFS/ECMWF)
inside the same worker — config change, no contract change.
